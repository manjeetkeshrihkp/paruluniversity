Chapter 5: Hosting & Deployment

This chapter explains how websites are published on the internet using domains, hosting services, and GitHub Pages.

1. Domains

A domain name is the address of a website on the internet. It is used to identify and access a website easily.

Example of Domain Names

  • www.google.com
  • www.github.com
  • www.collegewebsite.in

Parts of a Domain Name

  • Top-Level Domain (TLD) – .com, .org, .in
  • Second-Level Domain – google, github
  • Subdomain – www, blog

Domain names are registered through domain registrars.

2. Web Hosting

Web hosting is a service that stores website files on a server and makes them accessible on the internet.

Types of Hosting

  • Shared Hosting
  • VPS Hosting
  • Dedicated Hosting
  • Cloud Hosting

Hosting Example

When a user enters a domain name in the browser, the server delivers the website files to the user.

3. GitHub Pages

GitHub Pages is a free service provided by GitHub to host static websites directly from a GitHub repository.

Advantages of GitHub Pages

  • Free hosting
  • Easy to use
  • Direct GitHub integration
  • Best for student projects

Steps to Deploy Website Using GitHub Pages

  1. Create a GitHub repository
  2. Upload website files
  3. Go to Settings → Pages
  4. Select branch and save
  5. Access website via generated URL

Example URL

https://username.github.io/repository-name/

Summary

  • Domains provide website identity
  • Hosting stores website files
  • GitHub Pages allows free deployment
